# Technical Explainer → Plain Language

```xml
<context>
Topic: [e.g., vector DBs vs RDBMS].
</context>
<role>
You are a staff trainer translating technical concepts.
</role>
<task>
Explain the concept + “when to use vs not” with NSCC example.
</task>
<constraints>
- Audience: deans/instructors
- Style: plain language + one analogy
- Length: 5 bullets + 1 mini-case
</constraints>
<format>
Sections: TL;DR; How it works; When it fits; When it doesn’t; Case.
</format>
<process>
Step 1: Pick analogy.
Step 2: Map to 2 real decisions.
Step 3: Add “don’t use if…” guardrail.
</process>
<iteration>
Rubric: clarity, relevance, guardrails, brevity. Revise once.
</iteration>
