# UNIVERSE Bootstrap

**Purpose**: Build comprehensive story universe documentation - world bible, character database, continuity tracking for multi-story worlds.

---

## When to Use This Bootstrap

- Building story world for multiple stories
- Creating series universe (novel series, story collection)
- Need consistent world across works
- Tracking continuity for interconnected stories
- Developing story bible before writing
- Documenting existing universe retroactively

---

## Universe Building Workflow

### Phase 1: UNIVERSE DEFINITION
**Goal**: Establish scope and nature of universe

**Tasks**:

**Step 1: Universe Scope**
- **Name**: What do you call this universe?
- **Type**: 
  - Single-world universe
  - Multi-world/planetary system
  - Multiverse
  - Historical era
  - Alternate reality
- **Genre**: 
  - Fantasy
  - Science fiction
  - Horror
  - Contemporary with twist
  - Historical
  - Other
- **Tone**: Dark, hopeful, satirical, gritty, whimsical, etc.

**Step 2: Universe Rules**
- **What makes this universe unique?**
  - Magic system
  - Technology level
  - Supernatural elements
  - Historical divergence
  - Physics differences
- **What are the immutable rules?**
  - What always works this way
  - What's impossible
  - Fundamental laws
- **What can vary?**
  - What different stories can explore
  - Flexible elements
  - Story-specific additions

**Step 3: Story Scope**
- **How many stories?**
  - Novel series
  - Short story collection
  - Mix of formats
  - Ongoing universe
- **What connects them?**
  - Same world, different characters
  - Same characters, different times
  - Overlapping casts
  - Shared history/events
  - Thematic connections

**Deliverables**:
- `universe-overview.md` (scope and rules)

**Status Check**: ✅ Complete when universe defined

---

### Phase 2: WORLD-BUILDING
**Goal**: Create detailed, consistent world

**Tasks**:

**Step 4: Geography**
- **Physical world**:
  - Maps (if applicable)
  - Climate zones
  - Key locations
  - Natural resources
  - Landmarks
- **Inhabited areas**:
  - Cities/settlements
  - Nations/territories
  - Borders and boundaries
  - Travel routes
  - Dangerous areas

**Step 5: History**
- **Timeline**:
  - Major eras
  - Significant events
  - Turning points
  - Current era
- **Historical events**:
  - Wars/conflicts
  - Discoveries
  - Disasters
  - Social changes
- **Mythology/Legends**:
  - Origin stories
  - Folk tales
  - Prophecies
  - Cultural myths

**Step 6: Societies & Culture**
- **Social structures**:
  - Government types
  - Power hierarchies
  - Class systems
  - Social norms
- **Cultures**:
  - Ethnic/cultural groups
  - Languages
  - Religions/beliefs
  - Customs and traditions
  - Values and taboos
- **Institutions**:
  - Organizations
  - Schools/guilds
  - Military
  - Religious orders
  - Secret societies

**Step 7: Magic/Technology System**
- **How it works**:
  - Mechanics/rules
  - Limitations
  - Costs
  - Who can use it
- **Applications**:
  - Common uses
  - Advanced applications
  - Forbidden uses
- **History**:
  - How it developed
  - How it changed society
  - Current state

**Step 8: Economics & Daily Life**
- **Economy**:
  - Currency
  - Trade
  - Resources
  - Wealth distribution
- **Daily life**:
  - What do people eat?
  - How do they dress?
  - What do they do for fun?
  - Family structures
  - Education
  - Medicine/healing

**Deliverables**:
- `world-bible.md` (comprehensive world reference)
- `geography.md` (maps and locations)
- `history.md` (timeline and events)
- `cultures.md` (societies and customs)
- `magic-tech-system.md` (how things work)

**Status Check**: ✅ Complete when world fully documented

---

### Phase 3: CHARACTER DATABASE
**Goal**: Create cast that can span multiple stories

**Tasks**:

**Step 9: Major Characters**
- For each major character:
  - **Basic info**: Name, age, appearance
  - **Background**: History, family, origin
  - **Personality**: Traits, quirks, voice
  - **Motivation**: What drives them
  - **Abilities**: Skills, powers, weaknesses
  - **Role**: Protagonist, antagonist, supporting
  - **Story appearances**: Which stories they're in
  - **Character arc**: How they change over stories
  - **Relationships**: Connections to other characters

**Step 10: Supporting Cast**
- Recurring characters
- Minor characters who might reappear
- Character types/archetypes available
- Naming conventions

**Step 11: Character Continuity**
- **Timeline**: When each character exists
- **Knowledge**: What each character knows when
- **Relationships**: How they evolve
- **Status**: Alive/dead, location, condition
- **Cross-story tracking**: Appearances across works

**Deliverables**:
- `characters/` folder with individual character files
- `character-database.md` (master list)
- `character-timeline.md` (who's where when)

**Status Check**: ✅ Complete when cast documented

---

### Phase 4: CONTINUITY SYSTEM
**Goal**: Track events, timeline, and consistency

**Tasks**:

**Step 12: Master Timeline**
- Create chronological timeline:
  - Historical events (before stories)
  - Story events (from all works)
  - Future events (if known)
- Note:
  - When each story occurs
  - Major events and their dates
  - Character ages at different points
  - Cause and effect chains

**Step 13: Event Log**
- Track significant events:
  - What happened
  - When it happened
  - Who was involved
  - Consequences
  - Which stories reference it

**Step 14: Location Tracking**
- For each location:
  - Description
  - History
  - Current state
  - Which stories feature it
  - Changes over time

**Step 15: Object/Artifact Registry**
- Significant items:
  - Magic items
  - Technology
  - Historical artifacts
  - MacGuffins
- Track:
  - Properties
  - History
  - Current location
  - Appearances in stories

**Step 16: Mystery/Secret Tracking**
- **Universe secrets**:
  - What readers don't know yet
  - Reveals planned for future stories
  - Planted clues
- **Character secrets**:
  - Hidden backstories
  - Secret motivations
  - Identity reveals

**Deliverables**:
- `timeline.md` (master chronology)
- `event-log.md` (significant events)
- `locations.md` (place registry)
- `artifacts.md` (object tracking)
- `mysteries.md` (secrets and reveals)

**Status Check**: ✅ Complete when tracking systems established

---

### Phase 5: STORY INTEGRATION
**Goal**: Document how individual stories fit universe

**Tasks**:

**Step 17: Story Index**
- List all stories in universe:
  - Title
  - Format (novel, short, novella)
  - Timeline placement
  - Primary characters
  - Primary location
  - Key events
  - Reading order vs chronological order

**Step 18: Story Connections**
- Map relationships between stories:
  - Direct sequels/prequels
  - Parallel timelines
  - Shared characters
  - Overlapping events
  - Easter eggs and callbacks
  - Crossover potential

**Step 19: Continuity Verification**
- For each story:
  - Check against world bible
  - Verify timeline placement
  - Confirm character continuity
  - Ensure consistency with other stories
  - Note any retcons needed

**Step 20: Expansion Opportunities**
- Identify gaps:
  - Untold stories
  - Unexplored eras
  - Minor characters worth exploring
  - Locations to develop
  - Plot threads to continue

**Deliverables**:
- `story-index.md` (all stories)
- `story-connections.md` (how they relate)
- `expansion-ideas.md` (future story potential)

**Status Check**: ✅ Complete when all stories integrated

---

### Phase 6: UNIVERSE MAINTENANCE
**Goal**: Keep universe documentation current

**Tasks**:

**Step 21: Update Protocol**
- When writing new story:
  - Update timeline with events
  - Add new characters to database
  - Document new locations
  - Note new lore/world-building
  - Update continuity logs
  - Add story to index

**Step 22: Consistency Checking**
- Before publishing:
  - Verify against universe bible
  - Check character knowledge
  - Confirm timeline accuracy
  - Ensure magic/tech consistency
  - Review continuity with other stories

**Step 23: Retcon Management**
- If something needs changing:
  - Document the change
  - Note which stories affected
  - Update bible accordingly
  - Plan how to address in future works
  - Consider author's note if needed

**Step 24: Reader Resources**
- Consider creating:
  - Reading order guide
  - Character guide (spoiler-free)
  - World primer for new readers
  - Glossary
  - FAQ
  - Map/artwork

**Deliverables**:
- Maintained, current universe documentation
- Reader-facing materials (optional)

**Status Check**: ✅ Ongoing maintenance

## Working Principles

- **Bible is law**: Once documented, it's canon
- **Update immediately**: Document as you create
- **Track everything**: Small details matter
- **Be consistent**: Readers notice contradictions
- **Leave room**: Don't over-define, allow discovery
- **Make it accessible**: Reference must be usable
- **Version control**: Track changes to bible
